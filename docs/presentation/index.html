Что такое Hermes

Самостоятельный исполнитель сделок

Hermes - это десктопное приложение под Windows. Устанавливается двойным кликом, запускается из меню Пуск, работает в фоне. Подключается к торговому счёту оператора (MetaTrader 5, в работе - Binance/Bybit/OKX), сам открывает и закрывает сделки, пишет отчёт о каждом решении.

🔐

100% локально

Vault с ключами зашифрован Argon2id + Fernet. БД, логи, история - всё в %APPDATA%\BAI Core\Hermes\. Никаких облаков, никакой телеметрии без согласия.

🧠

Объясняет каждое действие

Перед каждой сделкой формируется markdown-отчёт: какие индикаторы согласились, какая стратегия сработала, уверенность. Никаких «чёрных ящиков».

🛡

Защита превыше прибыли

Три независимых circuit breaker'а. Watchdog следит за брокером. Кнопка аварийной остановки. Бот не сможет слить депозит за один плохой день.

Режимы торговли

Оператор выбирает сценарий

Два режима взаимоисключающие. Включается один из них или ни один (наблюдение). Лимит: 1-3 сделки в сутки, независимо от режима.

🛡 Проверенный сценарий

Одна откалиброванная стратегия

Только Trend Following с подтверждением ADX. Работает строго по 3-5 выбранным парам. Открывает только при уверенности ≥ 0.7. Это «торговать только то, что показало себя на истории».

  • Одна стратегия - Trend Following + ADX-фильтр
  • 3-5 настроенных пар, остальные игнорирует
  • Порог уверенности 0.7
  • Откалибровано через walk-forward backtest
🧠 Автономный режим

Развязанные руки

Все 4 стратегии в ансамбле с голосованием большинством. Сканирует все настроенные пары, выбирает одну с самым сильным сигналом - открывает там. Порог уверенности 0.5.

  • Trend Following + Momentum + Mean Reversion + Breakout
  • Любая пара из настроенных
  • Голосование: any / majority / all
  • Порог уверенности 0.5
Бот никогда не открывает позиции на нескольких парах одновременно. За один тик - максимум одна сделка на лучшем сигнале.
Один tick = одна минута

Что бот делает каждые 60 секунд

Цикл работы. Прерывается только командой оператора или Risk Engine'ом. Один tick - один цикл всех шагов.

  1. 1
    Сбор данных
    MT5 watchdog (3 фейла = пауза). OHLCV для каждой пары - 300 свечей. Текущие цены и спред.
  2. 2
    Indicator panel
    19 индикаторов на каждой паре: RSI, MACD, BB, ATR, EMA 50/200, ADX, ±DI, Stochastic, Donchian, ROC.
  3. 3
    Strategy ensemble
    4 стратегии формируют сигналы Long/Short/Flat. Голосование majority → одно решение по символу с обоснованием.
  4. 4
    Risk engine
    Daily loss < 5%, drawdown < 10%, позиций ≤ 5. Если нарушено - вход запрещён, бот наблюдает.
  5. 5
    Order + broadcast
    Лот = 0.5% эквити (снап под volume_step). SL/TP на сервер брокера. WebSocket рассылает обновление UI.
Аналитический движок

19 индикаторов · 4 стратегии · безграничные комбинации

📊

Технические индикаторы

Считаются по 300 свечам на каждой паре каждый тик. IndicatorPanel - чистая функция, легко покрывается тестами и не привязана к брокеру.

RSI (Wilder) MACD line/signal/hist BB upper/middle/lower ATR + ATR% EMA 50 EMA 200 ADX +DI / -DI Stochastic %K / %D Donchian high/low ROC trend consensus
⚔️

Торговые стратегии

Каждая выдаёт сигнал с уверенностью 0-1 и markdown-объяснением. Ensemble объединяет голосованием.

▲
Trend Following
EMA50/EMA200 cross + ADX > 25 + ±DI confirmation
≋
Momentum
MACD histogram > 0 + Stochastic %K/%D cross
⇄
Mean Reversion
Bollinger touch + RSI extreme + Stochastic turn
⤴
Breakout
Donchian pierce + ATR-expansion фильтр

Дефолтный ансамбль - Trend + Momentum. Эта единственная пара показала положительное ожидание на synthetic walk-forward в backtest v1.0.22. Mean Reversion и Breakout доступны опционально через UI.

Risk Engine

Три независимых circuit breaker'а

Каждый сторожит свою метрику. Сработал любой - новые входы блокируются, старые позиции остаются под брокерскими SL/TP. Оператор видит причину в Дашборде.

-5%
за сессию

Daily Loss Limit

Если эквити упало на 5% от стартового за день - бот перестаёт открывать. Сбрасывается в полночь UTC.

-10%
от пика

Drawdown Limit

От максимального эквити сессии. Защита от затяжной просадки - не даёт ансамблю усугубить положение.

5
max позиций

Max Open Positions

Одновременно открыто не более 5 позиций. Защита от перегрева портфеля - новые входы блокируются.

🎯

Broker-side SL/TP

Стопы и таргеты уходят на сервер брокера вместе с ордером (2×ATR / 4×ATR). Закрытие не зависит от состояния бота.

🐶

MT5 Watchdog

Каждый tick - health probe MT5. 3 фейла подряд = пауза + Telegram-уведомление «Брокер недоступен».

🛑

Kill Switch

Большая красная кнопка в UI закрывает все позиции по рынку немедленно и переводит бота в наблюдение.

Адаптивная калибровка

Еженедельный переподбор параметров

Рынок меняется. Параметры стратегий должны меняться вместе с ним - автоматически, безопасно, с возможностью отката.

01

Регим-детектор

Каждые 5 минут - режим каждой пары: trend / flat / high_vol

02

Сбор данных

N-дневное окно котировок для обучения, (N+7)-дневное для теста

03

Walk-forward

Challenger с новыми параметрами против champion с текущими

04

Применение

Champion заменяется только если Sharpe-ratio challenger'а выше

05

Откат

Если за 24-48ч после калибровки P&L упал - кнопка «Откатить» в UI

Установка

Установка двойным кликом

📥

Первая установка

Скачиваете Hermes-Setup-1.0.40.exe с GitHub Releases (~110 МБ). Двойной клик → Inno Setup → «Установить». Без админских прав, в user-space.

После установки: запустить MT5 (если ещё не запущен) → открыть Hermes → подключить брокера в одной форме → стратегия → запуск.

🔄

Обновление без переустановки

Hermes сам проверяет новые версии в Releases. Уведомление в UI - один клик «Обновить» - новый установщик ставится поверх старого.

База данных, vault, ui-prefs сохраняются. Версии 1.0.31+ автоматически мигрируют пользователей с master-password на passwordless mode.
