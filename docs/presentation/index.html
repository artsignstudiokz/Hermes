Что такое Hermes

Не торговый сигнал-сервис.
Самостоятельный исполнитель.

Hermes — это десктопное приложение под Windows. Устанавливается двойным кликом, запускается из меню Пуск, работает в фоне. Подключается к торговому счёту оператора (MetaTrader 5, в работе — Binance/Bybit/OKX), сам открывает и закрывает сделки, пишет отчёт о каждом решении.

🔐

100% локально

Vault с ключами зашифрован Argon2id + Fernet. БД, логи, история — всё в %APPDATA%\BAI Core\Hermes\. Никаких облаков, никакой телеметрии без согласия.

🧠

Объясняет каждое действие

Перед каждой сделкой — markdown-отчёт: какие индикаторы согласились, какая стратегия сработала, уверенность. Это не «чёрный ящик».

🛡

Защита превыше прибыли

Три независимых circuit breaker'а. Watchdog следит за брокером. Кнопка аварийной остановки. Бот не сможет слить депозит за один плохой день.

Два режима торговли

Оператор выбирает сценарий

Два режима взаимоисключающие. Включается один из них или ни один (наблюдение). Лимит: 1-3 сделки в сутки, независимо от режима.

🛡 Проверенный сценарий

Одна откалиброванная стратегия

Только Trend Following с подтверждением ADX. Работает строго по 3-5 выбранным парам. Открывает только при уверенности ≥ 0.7. Это «торговать только то, что показало себя на истории».

  • Одна стратегия — Trend Following + ADX
  • 3-5 настроенных пар, остальные игнорирует
  • Порог уверенности 0.7
  • Откалибровано через walk-forward backtest
🧠 Автономный режим

Развязанные руки

Все 4 стратегии в ансамбле с голосованием большинством. Сканирует все настроенные пары, выбирает одну с самым сильным сигналом — открывает там. Порог уверенности 0.5.

  • Trend Following + Momentum + Mean Reversion + Breakout
  • Любая пара из настроенных
  • Голосование: any / majority / all
  • Порог уверенности 0.5
Бот никогда не открывает позиции на нескольких парах одновременно. За один тик — максимум одна сделка на лучшем сигнале.
Как принимается решение

Каждый тик — пять шагов

Tick запускается каждые 60 секунд. Если что-то в pipeline блокирует — бот пишет почему.

1

Watchdog

Проверка соединения с брокером. 3 фейла → пауза + Telegram alert.

2

Analysis

OHLCV → 19 индикаторов на каждом символе. Кэшируется per-bar.

3

Ensemble

Выбранные стратегии голосуют. Берётся пара с максимальной уверенностью.

4

Risk Gate

Daily loss / drawdown / max positions. Если tripped — отказ.

5

Execute

place_order с SL/TP. TradeRow сохранён, Telegram пуш отправлен.

Индикаторы и стратегии

19 индикаторов · 4 стратегии · безграничные комбинации

📊

Технические индикаторы

Считаются за один проход на каждом тике, на каждом символе. Кэшируются в IndicatorPanel.

RSI (Wilder) MACD 12/26/9 Bollinger 20/2σ ATR (14) EMA 50/200 ADX + ±DI Stochastic 14/3 Donchian (20) trend consensus atr_pct (scale-free)
⚙

Торговые стратегии

Каждая выдаёт сигнал с уверенностью 0-1 и русским объяснением. Ensemble комбинирует через голосование.

  • Trend Following
    EMA cross + ADX ≥ 22 + ±DI confirmation
  • Momentum
    MACD histogram + Stochastic %K/%D cross
  • Mean Reversion
    Bollinger touch + RSI extreme + Stochastic turn
  • Breakout
    Donchian pierce + ATR expansion
Дефолтный ансамбль = Trend + Momentum. Это единственная пара, прошедшая валидацию на synthetic 12-month backtest с положительным ожиданием. Mean Reversion и Breakout доступны опционально через UI.
Управление рисками

Три независимых circuit breaker'а

Каждый guard срабатывает независимо. Tripped state не сбрасывается переключением режима — только полночь UTC или ручной reset.

📉

Daily Loss Limit

5% от начального equity сессии. Если today's P&L ≤ −5% → mode переключается в OFF, новые входы блокируются, в Telegram уходит alert.

📈

Drawdown Limit

10% от пика equity. Если equity отскакивает от peak более чем на 10% → стоп. Защищает прибыль, не только депозит.

🚫

Max Open Positions

5 одновременных позиций. Защищает от каскадного нагружения брокера и от ситуации «все стратегии одновременно решили войти».

🎯

Broker-side SL/TP

Каждый ордер открывается со стопом и тейком, отправленными напрямую брокеру. Бот может упасть — позиция закроется сама. SL = entry − 2×ATR, TP = entry + 4×ATR (risk/reward 1:2).

🩺

MT5 Watchdog

На каждом тике health-probe с 8с таймаутом. Один фейл → silent reconnect. Три фейла подряд → пауза + Telegram alert. Восстанавливается прозрачно.

⛔

Kill Switch

Одна кнопка → закрытие всех открытых позиций мгновенно. Snapshot цен сделан ДО close, TradeRow корректно обновлён, Telegram alert уходит.

Опыт оператора

Премиум-интерфейс. Никакой пустоты.

🎬

Интерактивный тур

При первом запуске Hermes сам проводит оператора по 9 шагам: режимы, индикаторы, безопасность, история. С кинематографичными анимациями.

📡

Реал-тайм через WebSocket

Equity-кривая, баланс, позиции, сигналы — всё push-based. REST остался safety-net на 60 секунд. -82% сетевого трафика по сравнению с polling.

📱

Мобильный доступ

QR + PIN → ngrok-туннель → дашборд на телефоне. Не браузерная закладка — PWA с push-уведомлениями, иконкой, splash screen.

📨

Telegram нотификации

Bot token + chat_id один раз. Каждая открытая сделка — markdown-отчёт с обоснованием. Kill switch и broker_down — отдельные алерты.

📊

Per-mode P&L

Три карточки на дашборде: Проверенный, Автономный, Ручные. Оператор видит какой режим реально приносит P&L, какой — нет.

📦

Tax CSV + Log Bundle

Налоговый отчёт за год одним кликом (формат КГД РК / 1099-B / MiFID). Архив логов для поддержки — без vault, без credentials.

Адаптация к рынку

Каждое воскресенье — переподбор параметров

Walk-forward калибровка с Optuna. 90 дней OHLCV, разделение 70/30, 100 trials. Champion-challenger: новые параметры применяются только если обогнали текущие на out-of-sample с защитой от overfitting.

Регим-детектор

Для каждой пары на 4h таймфрейме за 30 дней определяет состояние рынка:

trend (ADX > 25) flat (ADX < 20) high_vol (ATR% > 80 percentile)

Глобальный режим — мажоритарное голосование по парам. Используется для policy overlay в Auto-режиме.

Champion vs Challenger

Защита от overfitting:

  • 1. Минимум 50 трейдов на out-of-sample, иначе reject
  • 2. score = sharpe_oos − 0.5 × |sharpe_is − sharpe_oos|
  • 3. Применяется только если score_new ≥ 1.05 × score_current
  • 4. Max DD challenger ≤ 1.2× от champion

Оператор видит всю историю калибровок на странице «Адаптация» — таблица, diff параметров, кнопка «Запустить сейчас» и «Откатить».

Установка и обновления

Двойной клик — и в работе

📥

Первая установка

Скачать Hermes-Setup-1.0.27.exe с GitHub Releases (~80 МБ). Двойной клик → русский wizard с лого BAI Core → за 30 секунд готово. Установка per-user в %LOCALAPPDATA%\Programs\Hermes — без UAC.

🔄

Обновление без переустановки

Inno Setup детектит существующую установку по фиксированному AppId, закрывает старый Hermes.exe (CloseApplications=force), подменяет файлы, перезапускает новую версию. Vault, БД, стратегии — сохраняются.

Update Service внутри приложения автоматически проверяет GitHub Releases каждый раз при открытии Settings. Если есть новая версия — баннер с release notes + кнопка «Скачать и установить».

Валидация и тестирование

Production-ready значит — проверено

Unit тесты vault, KDF, JWT, validator, regime detector, signal ensemble. Integration — broker adapters (mocked), risk engine, daily counter. E2E — smoke API на лайв backend через ASGI.

Backtest валидация стратегий

На каждом коммите запускается mini-backtest по синтетическим данным. Default ensemble (Trend + Momentum) обязан показать положительное ожидание — иначе CI красный, релиз заблокирован.

Walk-forward в продакшене

Champion-challenger валидация каждое воскресенье. Параметры применяются только при положительной разнице sharpe_oos с защитой от overfitting. Без этого — rollback к предыдущим.
